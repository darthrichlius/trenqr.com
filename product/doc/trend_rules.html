GESTION

  • Créer une Tendance depuis BRAIN. Après la création, l'utilisateur est rediriger vers la Tendance nouvellement créée.
  • Créer une Tendance depuis TRPG. Après la création, l'utilisateur est rediriger vers la Tendance nouvellement créée.
  • Modifier les paramètres d'une Tendance. La modification implique de modifier tous les autres paramètres en même temps au niveau de la couche métier.
  • Supprimer une Tendance depuis BRAIN.
  • Mettre à jour les données sur le nombre de Tendances qui m'appartiennent après une suppression de Tendance. Seulement si l'utilisateur est sur sa page TMLNR
  • Mettre à jour les données sur le nombre de Tendances qui m'appartiennent dans l'entête après une suppression de Tendance. Seulement si l'utilisateur est sur BRAIN - MyTrends
  • Supprimer les Articles situées dans TIMELINE quand la Tendance liée vient d'être supprimée. Seulement si l'utilisateur est sur sa page TMLNR

ABONNEMENTS

  • Se connecter à une Tendance depuis sa page TRPG
  • Arrêter l'abonnement à une Tendance depuis BRAIN
  • Mettre à jour le nombre de Tendances suivies dans l'entete. Seulement si lu'tilisateur est sur BRAIN - MyTrends.
  • Arrêter l'abonnement à une Tendance de sa page TRPG

ARTICLES

  • Ajouter un nouvel Article à une Tendance depuis BRAIN.
  • Mettre à jour les données de la Tendance chaque fois qu'une opération add/delete est effectuée sur un Article.
  • Afficher les premiers Articles d'une Tendance. Le nombre est de 5 par colonnes.
  • Afficher le nombre de d'Articles ajoutés par un Compte dans une Tendance. La donnée est directement disponible sur la représentation de l'Article.
  • Afficher les Articles ajoutés ulterieurement.
  • Afficher les Articles plus anciens.

NOTE : Le fait de laisser le titre à la fin de l'URL nous permet de rendre le plus lisible possible ce qui est le plus important pour l'utilisateur : le TITRE.

TITRE
  • L'équipe technique peut modifier le titre d'une Tendance sans préavis si il ne respecte pas les conditions de la plateforme.
  • Le titre d'une Tendance n'est pas unique. En effet, cela permet de rester dans l'esprit du système concurrentiel entre les différents utilisateurs. Cependant, cela complique les traitements en ce qui concerne les recherches mais tout est fait pour y remédier grace à la création de table de références.
  • L'utilisateur ne doit pas utiliser de façon abusives des mots faisant référence à des marques déposées. La plateforme se reserve le droit d'utilser des mesures de rétorsions en cas de violations répétées de cette règle. Ces mésures peuvent aller jusqu'au bannissement.
  • Le titre ne doit pas dépasser 100 cractères.
  • Les caractères spéciaux sont autorisés dans le titre.
  • Le titre ne peut pas être uniquement composé que de caractères spéciaux. Cependant, l'utilisateur ne doit pas utiliser abusivement des caractères spéciaux.
  • L'encodage du titre d'une Tendance est "utf-8".
DESCRIPTION
  • La description permet d'apporter des informations complémentaires sur la tendance. Elle complète le titre.
  • La description ne doit pas dépasser 200 cractères.
  • Les caractères spéciaux sont autorisés dans la description.
  • La description ne peut pas être uniquement composé que de caractères spéciaux. Cependant, l'utilisateur ne doit pas utiliser abusivement des caractères spéciaux.
  • L'encodage du description d'une Tendance est "utf-8".
CATEGORIE
ACCES
  • Les Tendances sont des "lieux" publics. L'accès y est autorisé à tout visiteur quelque soit son état de connexion.
PARTICPATION
  • Le créateur d'une Tendance peut choisir entre une particpation PUBLIC ou PRIVATE.
  • PUBLIC : N'importe qu'elle utilisateur qui est CONNECTE et qui est ABONNÉ à la Tendance peut y poster des publications.
  • PRIVATE : Seul le propriétaire de la Tendance peut ajouter des publications.
PUBLICITE
  • Les pages de Tendances contiennent des publicités lorsque l'utilisateur n'est pas connecté.
